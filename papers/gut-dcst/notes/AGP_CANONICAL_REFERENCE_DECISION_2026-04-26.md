# Canonical AGP Reference Decision

Date: 2026-04-26

## Decision

Use the **local-QZA SILVA AGP taxonomy/hierarchy** as the canonical AGP
reference for the gut-dCST manuscript, together with the **corrected
nearest-lineage-set frozen transfer algorithm**.

For the paper-facing hierarchy, use the stricter local-QZA AGP run:

- AGP hierarchy:
  `/Users/pgajer/current_projects/gut_microbiome/outputs/dcst_analysis/runs/2026-04-26-agp-silva-local-qza-absorb-depth4-n0_50_25_25_25`
- Frozen transfer root:
  `/Users/pgajer/current_projects/gut_microbiome/outputs/dcst_validation/frozen_agp_silva_local_qza_2026-04-26_n0_50_25_25_25`

The older PRIME-based AGP hierarchy and the older `frozen_agp` transfer family
should no longer be the primary manuscript source of truth.

## Why

The main reason is **taxonomy-universe compatibility**.

With the local-QZA SILVA AGP rerun, the AGP discovery taxonomy and the external
validation taxonomies are generated by the same DADA2/SILVA pipeline. That
makes set-based comparisons meaningful:

- if a label appears in both datasets, it denotes the same taxon concept;
- if a label is in `S1 - S2`, the clean interpretation is that the taxon was
  present or detected in dataset 1 but not detected in dataset 2;
- non-overlap is no longer confounded by incompatible taxonomy pipelines.

The older PRIME-based AGP taxonomy does not satisfy this property. There, some
apparent AGP-vs-validation label mismatches reflect taxonomy-construction drift
rather than biological absence or transfer failure.

For a methods paper focused on cross-cohort transferability, this is a major
interpretability problem. A validation failure should mean “the sample does not
fit the learned ecological structure,” not “the two taxonomies are not speaking
the same language.”

## What This Implies

### 1. The transfer problem becomes well posed

The paper can now interpret overlap, non-overlap, and deeper-lineage assignment
in one shared taxonomy universe.

### 2. The corrected transfer semantics should be used everywhere

The canonical frozen-transfer implementation is the corrected version defined in

- `/Users/pgajer/current_projects/linf/papers/gut-dcst/notes/AGP_ABSORB_TRANSFER_DEFINITION_2026-04-26.md`
- `/Users/pgajer/current_projects/linf/papers/gut-dcst/notes/AGP_ABSORB_TRANSFER_IMPLEMENTATION_AUDIT_2026-04-26.md`

This means transfer is interpreted as assignment to the **closest valid AGP
absorb lineage set at each nominal depth**, with terminal stable lineages
remaining valid at deeper nominal depths.

### 3. The stricter `n0` hierarchy is the better paper-facing default

The comparison note

- `/Users/pgajer/current_projects/linf/papers/gut-dcst/notes/N0_SCHEDULE_COMPARISON_2026-04-26.md`

supports using:

- primary paper-facing hierarchy: `50,25,25,25`
- finer-grained sensitivity hierarchy: `50,25,12,10`

This keeps the main AGP structure statistically more defensible while still
allowing a resolution-increasing sensitivity analysis if needed.

## Tradeoffs

This choice is **not** based on maximizing apparent signal strength at all
costs.

Relative to the older PRIME-based branch, the taxonomy-harmonized local-QZA AGP
reference can:

- coarsen some deeper structure,
- change the identity of the “best” deeper transferred state,
- and weaken some formerly sharper depth-3/4 findings.

However, those costs are outweighed by the fact that the new branch yields a
much more defensible transfer experiment.

## Policy For Manuscript Assets

All manuscript-facing downstream analyses that depend on:

- AGP absorb assignments,
- AGP follow-up label summaries,
- or AGP-derived frozen transfer outputs

should be regenerated from the canonical local-QZA SILVA AGP hierarchy and the
corrected transfer root above.

Legacy outputs from:

- `outputs/dcst_analysis/runs/2026-04-11-absorb-depthscan-adaptive`
- `outputs/dcst_validation/frozen_agp`

may still be retained for historical comparison or sensitivity notes, but they
should not remain the canonical source of truth for the paper.
